# COMP0215 RL Coursework — 4×4 Connect-4 DQN Agent

## Overview
A **Double Dueling DQN** agent trained via self-play to play Connect-4 on a 4×4 board (need 4 in a row to win). Built from scratch — no pre-trained weights.

## Project Structure
```
RL_CW/
├── game/
│   └── tictactoe4x4.py    # Game environment + RandomAgent
├── agent/
│   ├── network.py          # CNN + Dueling DQN architecture
│   ├── dqn_agent.py        # DQN logic (Double DQN, target net)
│   └── replay_buffer.py    # Uniform + Prioritised replay
├── weights/
│   └── model_weights.pth   # Trained weights (generated by train.py)
├── logs/
│   └── training_log.csv    # Training metrics
├── report/
│   └── report.md           # Technical report
├── tests/
│   └── test_game.py        # Unit tests
├── train.py                # Training entry point
├── evaluate.py             # Evaluation vs random / vs itself
├── predict.py              # predict(board_state) -> (x, y) API
├── plot_results.py         # Plot training curves
└── requirements.txt
```

## Training
```bash
# Full training (10,000 episodes, ~30-60 min on CPU)
python train.py

# Quick smoke test (100 episodes)
python train.py --smoke-test

# Continue from checkpoint
python train.py --resume --episodes 5000
```

## Evaluation
```bash
python evaluate.py --games 1000
```

## Using the Predict API
```python
import numpy as np
from predict import predict

# board: 0=empty, 1=you, -1=opponent (4x4)
board = np.zeros((4, 4))
row, col = predict(board)
print(f"Best move: ({row}, {col})")
```

## Algorithm Details
- **Architecture**: Dueling Double DQN with CNN backbone
- **Input**: (2, 4, 4) board state (two binary planes — own/opponent pieces)
- **Output**: Q-values for all 16 board positions
- **Training**: Self-play with experience replay (50k buffer)
- **Exploration**: ε-greedy, ε decays from 1.0 → 0.05 over training
